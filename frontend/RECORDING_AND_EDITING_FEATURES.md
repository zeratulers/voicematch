## 🔧 技术实现细节

### 录音组件技术栈
```typescript
// 主要API使用
- navigator.mediaDevices.getUserMedia() // 麦克风访问
- MediaRecorder API // 音频录制
- Web Audio API // 音频处理
- Blob/File API // 音频数据处理
- URL.createObjectURL() // 音频播放
```

### 音频处理配置
```typescript
// 录音质量配置
const audioConfig = {
  echoCancellation: true,    // 回声消除
  noiseSuppression: true,    // 噪音抑制
  sampleRate: 44100,         // 高采样率
}

// 编码配置
const mimeType = 'audio/webm;codecs=opus' // 高效音频编码
```

## 🎮 用户交互流程

### 变体创建流程
1. **选择音频源** → 录音 或 上传文件
2. **录制/选择音频** → 
   - 录音：开始→暂停/继续→停止→预览→确认
   - 上传：选择文件→验证→上传→确认
3. **填写信息** → 方言类型、说话人信息等
4. **创建变体** → 自动上传到服务器

### 患者分配编辑流程
1. **查看当前分配** → 显示当前使用的变体信息
2. **选择新变体** → 从现有变体中选择
3. **预览变体** → 播放选中变体的音频
4. **创建专属变体**（可选）→
   - 选择方言类型
   - 录音或上传音频
   - 添加说话人信息
   - 创建并自动选择
5. **保存更改** → 更新患者分配
